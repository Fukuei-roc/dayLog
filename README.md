# DayLog

DayLog 是一個運行在 Windows 11、本機單機使用、不依賴雲端的每日工作待辦與筆記工具。它採用接近 Terminal / TUI 的操作方式，強調快速啟動、穩定可靠、鍵盤高效率，以及純本機 Markdown 儲存。

## 專案介紹

DayLog 啟動後會直接開啟今天的工作頁面，資料儲存在 `data/daily/YYYY-MM-DD.md`。每天的例行任務會從 `templates/dailyRoutine.md` 複製，上一次工作天尚未完成或已取消的臨時任務會自動延續到今天。所有資料都保存在本機，Markdown 檔案可直接閱讀與手動編輯。

## 技術選擇理由

- Python 3.9+：在 Windows 11 上容易部署與執行。
- Python 標準函式庫：不依賴雲端、不依賴 Web、降低安裝成本。
- 自製 Console TUI：保留 Terminal 風格並縮短啟動時間。
- 原生輸入對話框：新增與編輯文字時使用 Windows 原生輸入視窗，提升中文輸入穩定性。
- Markdown 檔案儲存：人類可讀，備份與手動修正都容易。

## 安裝方式

1. 安裝 Python 3.9 或更新版本。
2. 進入專案目錄。
3. 不需要額外安裝第三方套件。

## Windows 11 執行方式

```powershell
python -m app
```

程式會自動建立今天的 Markdown 檔並直接進入今天頁面。

## 快捷鍵列表

- `Up / Down`：上下移動游標
- `z / x`：跳到最上方或最下方
- `J / K`：在同一層級中向上或向下調整順序（`J` 上移，`K` 下移）
- `h`：切換「只顯示未完成項目 / 顯示全部項目」
- `Tab`：增加縮排
- `b`：減少縮排
- `Shift+Tab`：減少縮排（若終端支援）
- `a`：新增同層任務
- `A`：新增子任務
- `n`：新增筆記
- `m`：快速筆記，直接新增到 `臨時任務` 最下方
- `e`：編輯目前任務或筆記文字
- `d`：刪除目前項目
- `Space`：在 `[ ]` 與 `[x]` 間切換
- `c`：在 `[ ]` 與 `[-]` 間切換
- `Left`：收合目前項目
- `Right`：展開目前項目
- `/`：輸入指令
- `Enter`：在編輯模式中儲存
- `q`：離開程式
- `Esc`：取消文字輸入或取消編輯

文字輸入補充：

- `n`、`a`、`A`、`e`、`/` 進入文字輸入時，會使用原生輸入視窗
- `m` 快速筆記也會使用同一套原生輸入視窗
- 中文輸入法可直接在原生輸入視窗中使用，穩定度比 console 內逐鍵輸入高

顯示過濾補充：

- 按 `h` 後，會隱藏 `[x]` 已完成與 `[-]` 已取消項目
- 若父任務不是 `[ ]`，其底下子項會一併隱藏
- 若父任務是 `[ ]`，則仍會顯示其底下同樣為 `[ ]` 的任務分支

## Markdown 格式說明

```md
# YYYY-MM-DD

## 每日例行任務
- [ ] 任務
  - [ ] 子任務
  - 純文字

## 臨時任務
- [ ] 任務
```

- 任務支援 `[ ]`、`[x]`、`[-]`
- 子任務使用兩個空白縮排
- 筆記使用 `- 文字`
- 檔案可直接手動編輯

## 每日檔案建立邏輯

啟動時會：

1. 取得今天日期
2. 檢查 `data/daily/YYYY-MM-DD.md`
3. 若不存在則建立新檔
4. 從 `templates/dailyRoutine.md` 複製每日例行任務
5. 從上一個存在的工作日檔案延續臨時任務
6. 補齊兩個固定區塊
7. 直接載入今天畫面

若檔案不存在、模板被修改、區塊缺失或 Markdown 損壞，系統會盡量補齊最小結構並保留原始內容，避免直接崩潰。

## 延續任務規則

會從上一個存在的工作日檔案中的 `臨時任務` 區塊延續：

- `[ ]` 未完成任務
- `[-]` 已取消但尚未刪除的任務

不會延續：

- `[x]` 已完成任務
- `每日例行任務`

只要任務還存在且不是 `[x]`，隔天就會再次出現在 `臨時任務` 中；按 `x` 完成或直接刪除後，後續新的一天就不再延續。若任務底下有筆記，會一起保留；已完成的子任務不會被帶到今天。

## 指令說明

- `/todo 內容`
  - 新增待辦到 `臨時任務`
- `n` 筆記輸入框中的 `/date`
  - 展開為今天日期，後面可直接接文字或加空白，例如 `/date開始處理` 或 `/date 開始處理` 都會變成 `2026-04-01 開始處理`
- `n` 筆記輸入框中的 `/time`
  - 展開為目前時間，後面可直接接文字或加空白，例如 `/time時間測試` 或 `/time 時間測試` 都會變成 `09:30 時間測試`

## 使用範例

1. 執行 `python -m app`
2. 游標停在 `臨時任務`，按 `a` 新增任務
3. 選到任務後按 `A` 新增子任務
4. 按 `n` 新增筆記
5. 按 `/` 輸入 `/todo 回覆客戶`
6. 按 `n` 輸入 `/date 開始整理`
7. 按 `n` 輸入 `/time 已回報主管`
8. 任務完成後按 `Space`
9. 選到任務或筆記後按 `e`，可用左右鍵移動游標直接修改文字，按 `Enter` 儲存

## 測試方式

```powershell
python -m unittest discover -s tests -v
```

測試涵蓋：

- 建立新檔
- 模板複製
- 以上一個工作日延續任務
- `/todo`
- 筆記巨集 `/date 文字`
- 筆記巨集 `/time 文字`
- 層級解析
- 單行編輯器的插入、刪除與游標移動

## 已知限制

- 目前提供的是單行就地編輯；若要做大段多行內容調整，直接編輯 Markdown 仍會更方便。
- `Shift+Tab` 在少數終端可能被占用；此時可改用 `b` 減少縮排。
- 舊格式中若仍存在 `延續的臨時任務` 與 `今日臨時任務`，啟動後會自動合併為新的 `臨時任務`。
- 對非常規 Markdown 的修復以「不崩潰、保留內容、補齊最小結構」為優先，顯示順序可能略有調整。
