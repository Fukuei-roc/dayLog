# DayLog

DayLog 是一個運行在 Windows 11、本機單機使用、不依賴雲端的每日工作待辦與筆記工具。它採用接近 Terminal / TUI 的操作方式，強調快速啟動、穩定可靠、鍵盤高效率，以及純本機 Markdown 儲存。

## 專案介紹

DayLog 啟動後會直接開啟今天的工作頁面，資料儲存在 `data/daily/YYYY-MM-DD.md`。每天的例行任務會從 `templates/dailyRoutine.md` 複製，昨天尚未完成的臨時任務會自動延續到今天。所有資料都保存在本機，Markdown 檔案可直接閱讀與手動編輯。

## 技術選擇理由

- Python 3.9+：在 Windows 11 上容易部署與執行。
- Python 標準函式庫：不依賴雲端、不依賴 Web、降低安裝成本。
- 自製 Console TUI：保留 Terminal 風格並縮短啟動時間。
- Markdown 檔案儲存：人類可讀，備份與手動修正都容易。

## 安裝方式

1. 安裝 Python 3.9 或更新版本。
2. 進入專案目錄。
3. 不需要額外安裝第三方套件。

## Windows 11 執行方式

```powershell
python -m app
```

程式會自動建立今天的 Markdown 檔並直接進入今天頁面。

## 快捷鍵列表

- `Up / Down`：上下移動游標
- `Tab`：增加縮排
- `Shift+Tab`：減少縮排
- `a`：新增同層任務
- `A`：新增子任務
- `n`：新增筆記
- `d`：刪除目前項目
- `Space`：在 `[ ]` 與 `[x]` 間切換
- `c`：在 `[ ]` 與 `[-]` 間切換
- `Left`：收合目前項目
- `Right`：展開目前項目
- `/`：輸入指令
- `q`：離開程式
- `Esc`：取消文字輸入

## Markdown 格式說明

```md
# YYYY-MM-DD

## 每日例行任務
- [ ] 任務
  - [ ] 子任務
  - 純文字

## 延續的臨時任務
- [ ] 任務

## 今日臨時任務
- [ ] 任務
```

- 任務支援 `[ ]`、`[x]`、`[-]`
- 子任務使用兩個空白縮排
- 筆記使用 `- 文字`
- 檔案可直接手動編輯

## 每日檔案建立邏輯

啟動時會：

1. 取得今天日期
2. 檢查 `data/daily/YYYY-MM-DD.md`
3. 若不存在則建立新檔
4. 從 `templates/dailyRoutine.md` 複製每日例行任務
5. 從昨天檔案延續未完成臨時任務
6. 補齊三個固定區塊
7. 直接載入今天畫面

若檔案不存在、模板被修改、區塊缺失或 Markdown 損壞，系統會盡量補齊最小結構並保留原始內容，避免直接崩潰。

## 延續任務規則

只會延續昨天以下兩區中狀態為 `[ ]`` 的任務：

- `延續的臨時任務`
- `今日臨時任務`

不會延續：

- `[x]`
- `[-]`
- `每日例行任務`

若未完成任務底下有筆記，會一起保留；已完成或已取消的子任務不會被帶到今天。

## 指令說明

- `/todo 內容`
  - 新增待辦到 `今日臨時任務`
- `/date`
  - 在目前區塊插入今天日期
- `/time`
  - 在目前區塊插入目前時間

## 使用範例

1. 執行 `python -m app`
2. 游標停在 `今日臨時任務`，按 `a` 新增任務
3. 選到任務後按 `A` 新增子任務
4. 按 `n` 新增筆記
5. 按 `/` 輸入 `/todo 回覆客戶`
6. 按 `/` 輸入 `/date`
7. 任務完成後按 `Space`

## 測試方式

```powershell

```

測試涵蓋：

- 建立新檔
- 模板複製
- 延續任務
- `/todo`
- `/date`
- `/time`
- 層級解析

## 已知限制

- 目前沒有提供編輯既有文字內容的快捷鍵，若要修改文字可直接編輯 Markdown。
- `Shift+Tab` 在少數終端鍵碼可能不同，建議使用 Windows Terminal 或 PowerShell。
- 對非常規 Markdown 的修復以「不崩潰、保留內容、補齊最小結構」為優先，顯示順序可能略有調整。
